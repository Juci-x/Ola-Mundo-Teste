Meu primeiro site repositório: Resumo sobre Git e GitHub

1. O que é Git?

Git é um sistema de controle de versão distribuído, usado para rastrear mudanças em arquivos de código-fonte durante o desenvolvimento de software. Permite que múltiplos desenvolvedores trabalhem no mesmo projeto sem interferir uns nos outros. Criado por Linus Torvalds em 2005.

2. O que é GitHub?

GitHub é uma plataforma de hospedagem de código-fonte que utiliza o Git para controle de versão. Oferece funcionalidades adicionais como issues, pull requests, e integração com ferramentas de CI/CD. Facilita a colaboração em projetos open source e privados.

3. Comandos Básicos do Git

  • git init: Inicializa um novo repositório Git.
  • git clone [url]: Clona um repositório existente para o seu computador.
  • git add [arquivo]: Adiciona mudanças ao índice (staging area).
  • git commit -m "[mensagem]": Grava as mudanças no repositório com uma mensagem descritiva.
  • git push: Envia as mudanças locais para o repositório remoto.
  • git pull: Atualiza o repositório local com as mudanças do repositório remoto.
  • git branch: Lista, cria ou deleta branches.
  • git checkout [branch]: Muda para uma branch específica.
  • git merge [branch]: Combina as mudanças de uma branch com a branch atual.

4. Conceitos Importantes

  • Repositório: Um diretório onde o Git rastreia as mudanças nos arquivos.
  • Branch: Uma linha de desenvolvimento independente, usada para desenvolver funcionalidades ou correções sem afetar a branch principal (geralmente main ou master).
  • Commit: Um ponto na história do projeto onde mudanças são salvas.
  • Merge: Combina as mudanças de diferentes branches.
  • Pull Request (PR): Uma solicitação para integrar mudanças de uma branch em outra, comum em plataformas como GitHub.

5. Boas Práticas

  • Commits Atômicos: Faça commits pequenos e focados em uma única mudança.
  • Mensagens de Commit Claras: Escreva mensagens descritivas e concisas.
  • Trabalhe com Branches: Use branches para desenvolver novas funcionalidades ou correções.
  • Sincronize Frequentemente: Faça git pull regularmente para manter seu repositório local atualizado.

6. GitHub Features

  • Issues: Rastreie tarefas, bugs e melhorias.
  • Pull Requests: Revise e discuta mudanças antes de integrá-las.
  • Actions: Automatize workflows de CI/CD diretamente no GitHub.
  • Wiki: Documente seu projeto com páginas de wiki.
  • Projects: Organize tarefas e milestones com quadros Kanban.
